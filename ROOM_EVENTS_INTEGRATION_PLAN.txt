Mystic Onslaught - Room Events Integration Plan (LOCKED 2026-07-10)
===================================================================
Companion to ROOM_INTERACTION_IDEAS.txt (the design doc). This file records
HOW the room-events system hooks into the CURRENT code, after the meta-loop /
village work of 2026-07-09. Read the design doc for fantasy/tables; read this
for architecture + decisions.

KEY REALITY CHECK vs the design doc
-----------------------------------
- The doc assumes decision rooms slot into RoomType::Rest. REST ROOMS NO LONGER
  EXIST. DungeonGen.cpp: "No Rest rooms in DungeonRun - healing is handled by
  the Store on entry." So slotting changed (see Step 3).
- Cursed Wager is now gated to the push-onward (double-or-nothing) boss path via
  _wagerAccessGranted (zone 2+). Risk Shrine and Cursed Wager therefore live in
  different places but SHARE the pending-modifier plumbing.
- Village return path is NOT hooked to the main game yet (deferred). Room
  specials live entirely inside DungeonRun and do not touch the village or the
  death->Poe->class cutscene.

WHAT ALREADY EXISTS (build on it)
---------------------------------
- DungeonRoomState (Engine.h ~1144): persists cleared/visited/eliteMechanic/
  survivors, survives re-entry. Room-special state rides this struct.
- Room-clear choke point: Engine.cpp ~730, _dungeonRoomStates[idx].cleared=true.
  Single place to tick modifier durations (addendum A6).
- Cursed Wager card UI: OpenCurseShrine/UpdateCurseShrine/DrawCurseShrine
  (Engine.cpp ~9111+). Debugged kbd+gamepad+mouse+affordability. -> extract (A8).
- DrawModifierStack() (Engine.h ~1164): top-right pill lane already comments
  "affix / wager / future curses". The curse HUD lane is pre-anticipated.
- QueueRoomIntroBanner(): special-room intro banner already exists.

DECISIONS (locked with Robert 2026-07-10)
-----------------------------------------
- Slotting: TAG STANDARD ROOMS (add specialType to room state; no new RoomType).
- Frequency: 1 guaranteed + 1 at ~50% decision room per dungeon (~12 rooms).
- Card UI: REFACTOR-FIRST. Extract ChoiceCardScreen, route Cursed Wager through
  it with zero visual change, THEN build Risk Shrine as consumer #2.
- ChoiceCardScreen lives as Engine methods + a state struct for now (matches
  where the wager lives; no vcxproj churn). Promote to its own file when a
  second/third room consumes it.
- First room = Risk Shrine.

BUILD ORDER
-----------
P1 FOUNDATION (build once, verify, then rooms are cheap content)
  Step 1  Extract ChoiceCard + ChoiceCardScreen from the wager UI. Wager = first
          consumer, ZERO visual change. Build + verify identical.
  Step 2  RoomSpecialType enum (GameTypes.h): None, RiskShrine, CursedReward,
          OmenProphecy, LockedArmory, MutationAltar, RelicForge.
  Step 3  Extend DungeonRoomState: specialType / specialClaimed / specialChoice
          + option data. Roll at GENERATION time seeded dungeonSeed*31+roomIdx
          (A7: dailies identical, re-entry can't reroll). Tag 1 guaranteed + 1
          ~50% Standard rooms. Draw "?" minimap marker until visited.
  Step 4  Unified pending-modifier list: one struct {label, enemyDmgMult,
          enemyHpMult, enemySpeedMult, rewardGoldMult, rewardXpMult,
          durationRooms, bonusGold/Xp accumulators}. Feed DrawModifierStack.
          Tick durations at Engine.cpp:730 (room cleared). Cap = 2 accepted
          (A4). Death clears all (A9).

P2 FIRST CONTENT
  Step 5  Risk Shrine room. Three next-combat-room contracts:
            Greed  - next room gold x2, enemies +25% damage
            Hunger - next room XP x2,  enemies +25% move speed
            Pride  - next room spawns an elite miniboss, reward chest upgrades
          Via ChoiceCardScreen. Resolve toast (A2, reuse "+N Echoes banked"
          pattern). Enemy spawn-tint tell (A3). Debug force keys.

DURATION SEMANTICS (A6, write once, apply everywhere)
-----------------------------------------------------
- Durations tick ONLY on combat-room completion (the cleared=true event).
- Shop/decision/empty rooms never tick.
- "Next room" = next COMBAT room entered (contract stays armed through detours).
- Boss rooms count as combat rooms.
- Death clears all pending + active modifiers. No carryover.
- Accepted-modifier cap = 2 (shrine contracts + reward curses + omens only;
  wager/affix/ascension do NOT count).
